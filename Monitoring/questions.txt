Section 2: Monitoring Tools Exploration (20 pts)
5. What is Prometheus used for in monitoring? (5 pts)
Open-source monitoring system built around time-series metrics. Works on a pull-based model — periodically scrapes /metrics endpoints exposed by apps/exporters, stores data in its own time-series database, and lets you query it with PromQL. Includes built-in alerting via Alertmanager. De facto standard for monitoring in Kubernetes environments.
6. Describe how Grafana complements Prometheus. (5 pts)
Prometheus is strong at collection, storage, and querying, but its own UI is minimal. Grafana connects to it as a data source and adds rich visual dashboards, correlation with other data sources (e.g. logs via Loki) in the same view, and centralized visual alerting. In short: Prometheus = data engine, Grafana = visualization/exploration layer.
